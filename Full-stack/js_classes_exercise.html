Some “true ;)” exercises to practice working with classes in JavaScript:

  1. Basic Class Definition: Create a class named Car with properties make, model, and year. Add a method called displayInfo that prints out the car’s information.

  2. Constructor and Instance Methods: Extend the Car class. Add a constructor that initializes the properties. Include an instance method named startEngine that logs a message indicating the engine has started.

  3. Inheritance: Create a class SportsCar that extends the Car class. Add a property topSpeed to the SportsCar class. Override the displayInfo method to include the top speed.

  4. Encapsulation1: Modify the Car class to include a private property for the year property. The getYear method should return the year, and the setYear method should validate that the year is a positive number.

  5. Encapsulation2: Create a class Race that has a private property participants (an array of Car instances). Add a method addParticipant to add a car to the race.

  6. Polymorphism: Implement a method called race in both the Car and SportsCar classes. The implementation should log a message specific to each class.

Possible solutions

  1. Basic Class Definition:
class Car {
      constructor(make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
      }
    
      displayInfo() {
        console.log(`${this.year} ${this.make} ${this.model}`);
      }
}
  1. Constructor and Instance Methods:
class Car {
      constructor(make, model, year) {
        this.make = make;
        this.model = model;
        this.year = year;
      }
    
      displayInfo() {
        console.log(`${this.year} ${this.make} ${this.model}`);
      }
    
      startEngine() {
        console.log('Engine started!');
      }
}
  1. Inheritance:
class SportsCar extends Car {
      constructor(make, model, year, topSpeed) {
        super(make, model, year);
        this.topSpeed = topSpeed;
      }
    
      displayInfo() {
        super.displayInfo();
        console.log(`Top Speed: ${this.topSpeed} mph`);
      }
}

By calling super.displayInfo();, you are essentially saying, “First, execute the displayInfo method of the parent class (Car), and then execute the additional code I’ve written in the displayInfo method of the SportsCar class.” This ensures that you retain the functionality of the parent class while extending it in the child class.

  1. Encapsulation 1:
class Car {
     constructor(make, model, year) {
       this.make = make;
       this.model = model;
       this.#year = year;
     }
   
     getYear() {
       return this.#year;
     }
   
     setYear(value) {
       if (value > 0) {
         this.#year = value;
       } else {
         console.log('Invalid year');
       }
     }
   
     displayInfo() {
       console.log(`${this.#year} ${this.make} ${this.model}`);
     }
}
  1. Encapsulation 2:
class Race {
     constructor() {
       this.#participants = [];
     }
   
     addParticipant(car) {
       this.#participants.push(car);
     }
   }
  1. Polymorphism:
class Car {
     // ... (previous code)

       console.log('Car is racing!');
     }
   }

     // ... (previous code)

       console.log('Sports car is racing at top speed!');
     }
  }
